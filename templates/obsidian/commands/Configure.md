# Configure

Use this command when the user defines or updates project conventions.

## Input

User-provided categories, priorities, project areas, repository references, examples, or GitHub-derived suggestions.

## Procedure

1. Read `project/config.md`.
2. Preserve user-provided configuration as authoritative.
3. Record GitHub-derived categories, labels, areas, and repository mappings as tentative until the user confirms them.
4. Keep secrets out of tracked files.
5. Ensure every category has `key`, `description`, and at least one example.
6. Ensure every priority has `key`, `description`, and at least one example.
7. Update `project/log.md`.
8. Run the `Check` procedure.

## Output

Concise summary of confirmed configuration, tentative suggestions, and check result.
