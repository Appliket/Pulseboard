# Update

Use this command when the user changes a task state, scope, criteria, notes, dependencies, or review outcome.

## Procedure

1. Locate the task by id, path, or wikilink.
2. Update task frontmatter and body while preserving evidence history.
3. Move the Kanban card to the lane matching task status.
4. Split broad tasks or merge duplicates when evidence supports it.
5. Add follow-up tasks when review/check findings expose missing work.
6. Update graph artifacts.
7. Update `project/log.md`.
8. Run the `Check` procedure.

## Status Mapping

- `todo` -> `## To Do`
- `in-progress` -> `## In Progress`
- `in-review` -> `## In Review`
- `done` -> `## Done`
